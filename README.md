# Ahrimod

Телеграм-бот модерации для чата сервиса **AhriVPN**.

## Что умеет

- Модерация ссылок и стоп-слов в защищённых чатах
- Whitelist / blacklist доменов с поддержкой wildcard (`*.example.com`)
- Стоп-слова с защитой от обходов (гомоглифы, zero-width, пробелы между буквами, транслитерация)
- Сигнатуры сообщений: точные SHA256 + нечёткие SimHash
- Ручная модерация неизвестных ссылок/слов через инлайн-кнопки в админ-чате
- Разная логика для новых и доверенных пользователей:
  - **Новый**: ссылка/слово неизвестно → удалить + на ревью; известный blacklist → бан/мут
  - **Доверенный**: ссылка/слово неизвестно → НЕ удалять, уведомить модератора; blacklist → удалить + варн
- CAS (Combot Anti-Spam) проверка при входе
- Бан за ссылки/упоминания каналов в имени пользователя
- Команды модераторов для управления списками
- Audit log всех действий

## Локальный запуск (для разработки)

```bash
cd ahrimod
cp .env.example .env
# отредактировать .env
python main.py
```

`main.py` сам поставит зависимости при первом запуске.

## Установка на Ubuntu сервере (production)

См. подробную пошаговую инструкцию в **`INSTALL.md`** — установка под отдельным пользователем, запуск через systemd с автостартом при перезагрузке и автоматическим перезапуском при сбоях.

## Настройка

Открыть `.env` и заполнить:

| Поле | Что |
|---|---|
| `BOT_TOKEN` | Токен от @BotFather |
| `ADMIN_CHAT_ID` | ID приватного чата куда уходят сообщения на ревью |
| `LOG_CHAT_ID` | ID чата для audit-лога (можно тот же) |
| `PROTECTED_CHAT_IDS` | ID чатов которые модерируем (через запятую) |
| `ADMIN_USER_IDS` | ID админов (через запятую) |
| `TRUST_MIN_MESSAGES` | Сколько "квалифицирующих" сообщений нужно для доверия (по умолчанию 3) |
| `TRUST_MIN_INTERVAL_MINUTES` | Минимальный интервал между квалифицирующими сообщениями (по умолчанию 60) |
| `TRUST_MIN_HOURS` | Сколько часов ждать после набора порога (по умолчанию 24) |
| `NEW_USER_PUNISHMENT` | `ban` или `mute` для новых юзеров |

⚠️ Бот должен быть **админом** в защищённых чатах с правами:
- Удалять сообщения
- Банить пользователей
- Ограничивать пользователей

⚠️ В BotFather нужно отключить privacy mode (`/setprivacy` → Disable), иначе бот не увидит обычные сообщения в группах.

## Команды модератора

| Команда | Описание |
|---|---|
| `/help` | Список команд |
| `/addbanword слово` | Добавить стоп-слово |
| `/rmword слово` | Удалить слово |
| `/addgooddomain domain` | Whitelist домена |
| `/addbandomain domain` | Blacklist домена (поддерживает `*.example.com`) |
| `/rmdomain domain` | Удалить домен |
| `/listdomains` | Список доменов |
| `/trust` (реплай) | Пометить юзера доверенным навсегда |
| `/untrust` (реплай) | Снять флаг доверия |
| `/ban` (реплай) | Забанить юзера + удалить сообщение |
| `/unban` (реплай или id) | Разбанить |
| `/info` (реплай) | Досье на юзера |
| `/addsignature` (реплай) | Добавить сообщение в базу сигнатур |
| `/stats` | Статистика |

## Как работает доверие

Чтобы стать "доверенным", пользователь должен пройти три проверки подряд:

1. **Квалифицирующие сообщения**. Каждое сообщение пользователя без ссылок и стоп-слов может стать "квалифицирующим", если с предыдущего квалифицирующего прошло не меньше `TRUST_MIN_INTERVAL_MINUTES` (по умолчанию 60 минут). Это защита от накрутки счётчика спамом подряд.
2. **Накопленный порог**. Нужно набрать `TRUST_MIN_MESSAGES` (по умолчанию 3) квалифицирующих сообщений. В момент набора фиксируется отметка времени `qualified_at`.
3. **Период ожидания**. После момента `qualified_at` должно пройти `TRUST_MIN_HOURS` часов (по умолчанию 24).

При значениях по умолчанию: новый пользователь не может стать доверенным быстрее чем за **24 часа** после третьего сообщения, и сами три сообщения должны быть растянуты минимум на **2 часа** (по часу между каждым). То есть быстрейший возможный путь к доверию — около 26 часов.

Модератор может в любой момент дать доверие вручную через `/trust` (реплай) или забрать через `/untrust`. Команда `/info` показывает текущий прогресс пользователя.

## Логика модерации (вкратце)

1. Любое сообщение от **админа** пропускается
2. Если совпала **сигнатура** забаненного сообщения → бан + удаление
3. Извлекаются ссылки и стоп-слова. Если ничего нет → +1 к счётчику и пропустить
4. Если найден **запрещённый домен**:
   - Новый юзер → бан/мут + удаление
   - Доверенный → удаление + варн
5. Если найдено **стоп-слово** у нового юзера → бан/мут + удаление
6. Если все ссылки в whitelist и стоп-слов нет → пропустить
7. Иначе — на ручную модерацию:
   - Новый → удалить + отправить в админ-чат
   - Доверенный → не удалять, уведомить админ-чат
